## What to build

Suppress `Schätzung` notes and estimate confirmation affordances for values that are empty, missing, or effectively `0`.

These notes are useful when the app pre-fills a plausible estimate. They become confusing when shown below blank fields or placeholder zeroes because the user reads them as a real model estimate.

## Acceptance criteria

- [ ] Empty fields do not show a `Schätzung` note or confirmable estimate badge.
- [ ] Placeholder/default `0` values do not show a `Schätzung` note unless that zero is a real domain value the user intentionally entered.
- [ ] Once the user enters a non-empty estimated value, existing evidence UI still behaves normally.
- [ ] Confirming a real estimate still promotes the relevant `evidenceMap` entry.
- [ ] Regression coverage includes at least one onboarding product field with an empty value and one with a zero value.
