# Issue tracker: GitHub

Issues, PRDs, follow-ups, and triage state for this repo live in GitHub Issues:

- Repository: `PeterHartwieg/Rentenrechner`
- Canonical tracker URL: `https://github.com/PeterHartwieg/Rentenrechner/issues`

Use the GitHub connector when it is available in Codex. Otherwise use the `gh`
CLI from inside the repo clone; it can infer the repository from `git remote -v`.

## Conventions

- Create one GitHub issue per independently actionable bug, enhancement, PRD, or
  implementation slice.
- Apply the appropriate triage labels from `docs/agents/triage-labels.md`.
- Put long implementation briefs, reproduction notes, and design context in the
  issue body or comments, not in local markdown issue files.
- During triage, every issue body or comment generated by an AI triage workflow
  must start with:

  ```markdown
  > *This was generated by AI during triage.*
  ```

## Common operations

- **Create an issue:** use the GitHub connector's create-issue action, or
  `gh issue create --title "..." --body-file <file> --label needs-triage`.
- **Read an issue:** use the GitHub connector's fetch/view action, or
  `gh issue view <number> --comments`.
- **List triage queue:** filter open issues by `needs-triage`, `needs-info`, or
  missing labels.
- **Apply labels:** use the GitHub connector's label action, or
  `gh issue edit <number> --add-label "needs-triage"`.
- **Comment:** use the GitHub connector's comment action, or
  `gh issue comment <number> --body-file <file>`.

## Historical local files

`.scratch/` still contains historical PRDs, old local issue files, temporary
working notes, screenshots, and migration context. Treat those files as
read-only background unless the user explicitly asks to edit or archive them.

Do not create new `.scratch/<feature>/issues/*.md` files for tracker work.

## When a skill says "publish to the issue tracker"

Create a GitHub issue in `PeterHartwieg/Rentenrechner` and apply `needs-triage`
unless the user asked for a different state.

## When a skill says "fetch the relevant ticket"

Fetch the GitHub issue by number or URL, including comments and labels.
