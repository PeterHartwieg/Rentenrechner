# llms.txt and llms-full.txt

**Status: experimental.** These two files are published at the site root as a
discovery convenience for AI assistants. They are **not** a measured success
surface and are **not** a guaranteed citation channel — neither the format nor
the consumer ecosystem is standardised yet. Treat them as a low-cost
experiment that we revisit each statutory-update cycle.

Spec reference: <https://llmstxt.org>

## Files

| Path | Generated by | Purpose |
|------|--------------|---------|
| `dist/llms.txt` | `generateLlmsTxt()` in `src/seo/llmsTxt.ts` | Site name, one-paragraph purpose, and one bullet per in-sitemap route with title + absolute canonical URL + summary. |
| `dist/llms-full.txt` | `scripts/prerender.mjs` (after SSG) | Full prose body per in-sitemap, non-404 route. HTML is read from the prerendered output and stripped via a simple `/<[^>]+>/g` regex, with `<script>` / `<style>` content removed first and whitespace collapsed. Sections are headed by `# <title>` + `URL:` + `Stand:` and separated by `---`. |

Both files are written from the build pipeline (`vite build` →
`node scripts/prerender.mjs`) at the same step that emits `sitemap.xml` and
`robots.txt`. They are deterministic: the same registry plus identical
prerendered HTML produces byte-identical files.

## What is excluded

- The `/404` route (consistent with `sitemap.xml`).
- Share-state URLs (`?s=...`) and topic-preselection links (`?topic=...`).
  Canonical URLs only — same exclusion rule as the sitemap.
- Any route flagged `inSitemap: false` in `publicRouteRegistry.ts`.

## What is **not** done

- **No sitemap entry.** `llms.txt` and `llms-full.txt` are not web pages and
  are not added to `sitemap.xml` (locked decision in issue #10).
- **No user-facing promotion.** There is no in-app link, banner, or footer
  reference to either file. Crawlers that recognise the convention will find
  them at the site root by their well-known names.
- **No robots.txt change.** Robots policy is owned by issue #02 and is not
  modified here.
- **No measurement.** We do not track AI citations or referrals from these
  files — the project is privacy-first and has no analytics/telemetry.

## Why experimental

The `llms.txt` proposal is young (2024/2025) and adoption among AI
assistants varies. Publishing the file is essentially free for us once the
build wiring exists, but we make no claim that consumers honour it, nor that
inclusion improves citation rates. If the convention is abandoned upstream,
we remove the build step in one commit — no other surface depends on these
files.

## When to update

- After every statutory-year refresh (e.g. 2026 → 2027): the `Stand:` lines
  in `llms-full.txt` are sourced from each route's `dateModified` field in
  `publicRouteRegistry.ts`. Update the registry; the build picks it up.
- After adding a new public route: add the registry entry and the file
  picks up the new bullet automatically. No edit to `llmsTxt.ts` or the
  prerender script is required.
